Что мы изучили?

Модели сетей

  • ISO/OSI: 7 уровней (физический, канальный, сетевой, транспортный, сеансовый, уровень представления, прикладной).
  • TCP/IP: реальная модель, 4 уровня:
    • Канальный (Ethernet) — передача данных по проводам или Wi-Fi.
    • Сетевой (IP) — отвечает за адресацию (как почтовые индексы).
    • Транспортный (TCP/UDP) — гарантирует доставку данных.
    • Прикладной (HTTP, FTP) — сайты, почта, мессенджеры.

Адреса в сети

  • MAC-адрес: уникальный серийный номер устройства.
  • IP-адрес: например, 192.168.1.1.
  • Порт: как номер квартиры в доме, например, 80 для сайтов.

TCP — надёжная доставка

Работает через трёхстороннее рукопожатие:

  1. Компьютер A: "Привет, давай общаться?" (SYN)
  2. Компьютер B: "ОК, готов!" (SYN-ACK)
  3. Компьютер A: "Супер, начинаю!" (ACK)

Если данные теряются — TCP отправляет их повторно.

NAT — "общий интернет"

Позволяет нескольким устройствам использовать один внешний IP-адрес.

  • Пример: Телефон (192.168.1.2) → Роутер (88.8.8.8) → Интернет
  • CVE — база данных известных уязвимостей.
  • CVSS — стандарт оценки опасности уязвимостей.
  • CWE — перечень типовых ошибок в коде.
  • Сканирование — поиск слабых мест в системах.
  • Банк данных угроз — хранение информации о возможных атаках.
  • Поиск уязвимостей — активное выявление проблем.
  • Причины возникновения уязвимостей:
    • Ошибки дизайна
    • Ошибки в коде
    • Неправильные настройки конфигурации
    • Человеческий фактор
  • Виртуальная память: программы работают с виртуальными адресами, ОС переводит их в физические.
  • Кэш-память: быстрый доступ к данным процессора:
    • L0 — самая быстрая
    • L1, L2 — чуть медленнее
  • Прерывания: сигналы, прерывающие выполнение задач.
  • Процессы: программы со своими инструкциями и памятью. Если два процесса одинаковые, данные копируются только при изменении (Copy-on-Write).
  • Файловая система:
    • Inode — информация о файле (размер, права, расположение)
    • Buffer cache — ускоряет доступ к часто используемым данным
  • Обработка ошибок: ОС следит за состоянием системы и останавливает неполадки.
  • Red team: атакующие (Black box / White box)
  • Blue team: защитники (SOC — центр мониторинга)
  • Yellow team: эксперты по архитектуре и политикам безопасности
  • Purple team: совместная работа Red + Blue
  • Green team: Yellow + Blue — оптимизация защиты
  • Orange team: Red + Yellow — создают целевые сценарии атак
  • Угрозы:
    • Фишинг: обманные письма/сайты для кражи паролей
    • DDoS-атаки: перегрузка сервера большим количеством запросов
  • Средства защиты:
    • Межсетевые экраны: фильтруют входящий и исходящий трафик
    • Сканирование: ScanOval, ScamSAS, OpenSAS
  • Резервные копии: Cyber Backup
